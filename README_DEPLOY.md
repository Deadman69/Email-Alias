# EmailAlias — Deployment

Production deployment guide.

## Requirements

| Item | Detail |
|---|---|
| Server | Linux, **port 25 open** (inbound), 512 MB RAM min |
| Runtime | Docker ≥ 24 with Compose |
| Domain | DNS access for an MX record |
| HTTPS | Handled automatically by Caddy (Let's Encrypt) |

---

## 1 — Environment variables

Copy `.env.example` to `.env` and fill in the required values:

```env
# ── Application ───────────────────────────────────────────────────────────────
APP_KEY=                          # generated by: php artisan key:generate
APP_URL=https://mail.company.com
# Domains are configured in Settings → Domains after first login (no APP_DOMAIN env var)

# ── Database ──────────────────────────────────────────────────────────────────
DB_PASSWORD=<strong_secret>

# ── SMTP receiver ─────────────────────────────────────────────────────────────
SMTP_INTERNAL_SECRET=<strong_secret>   # shared between smtp-server and Laravel

# ── WebSocket (Reverb) ────────────────────────────────────────────────────────
REVERB_APP_KEY=<strong_secret>
REVERB_APP_SECRET=<strong_secret>
```

> All business settings (SSO, 2FA, limits, retention…) are managed exclusively via `/admin/settings` after first launch. Sensitive secrets (e.g. Azure client secret) are stored encrypted in the database.

---

## 2 — DNS

Add an MX record on your domain:

```
Type      MX
Name      @   (or mail.company.com)
Value     <server IP or hostname>
TTL       300
Priority  10
```

Verify: `nslookup -type=MX mail.company.com`

---

## 3 — Launch

```bash
# Start all services
docker compose up -d

# First-time setup
docker compose exec app php artisan key:generate
docker compose exec app php artisan migrate --force

# Create the first Super Admin
docker compose exec app php artisan admin:create --super-admin
```

---

## 4 — Initial configuration (UI)

Log in at `https://<APP_URL>/admin/settings` with the Super Admin account.

Configure in order:
1. **General** — application name, default language
2. **Authentication** — enable/configure Azure AD SSO, disable local login if SSO-only
3. **Security** — enforce 2FA if required
4. **Aliases / Email** — adjust limits and retention as needed

---

## 5 — Verification

```bash
# Container status
docker compose ps

# Live logs
docker compose logs -f app smtp-server

# Test SMTP ingestion (replace address with an alias created via the UI)
curl -s -X POST http://localhost:9000/internal/inbound \
  -H "Content-Type: application/json" \
  -H "X-SMTP-Secret: <SMTP_INTERNAL_SECRET>" \
  -d '{"to":["alias@mail.company.com"],"from_address":"test@example.com","from_name":"Test","subject":"Test","body_text":"Hello","body_html":null,"headers":{},"size_bytes":5}'
```

## Internal services & ports

| Service | Role | Internal port |
|---|---|---|
| `app` | Laravel (PHP-FPM) | 9000 |
| `smtp-server` | SMTP ingestion | 25 |
| `reverb` | WebSocket | 8080 |
| `worker` | Queue jobs | — |
| `scheduler` | Cron (cleanup) | — |
| `db` | PostgreSQL 16 | 5432 |
| `caddy` | HTTPS reverse proxy | 80 / 443 |

Only ports 80, 443, and 25 are exposed publicly. Everything else stays on the internal Docker network.
